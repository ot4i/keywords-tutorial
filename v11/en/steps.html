Using custom keywords with message flows

Import projects

When you click Import, two projects will be created in your workspace: An application named ExampleKeywords and a Java project name ExampleKeywordsJava.

ExampleKeywords provides a single message flow named Keywords. The message flow provides an HTTPInput node wired to a JavaCompute node, then an HTTPReply. The HTTPInput node is configured to receive any input message as BLOB. The JavaCompute node creates an XML output message. The output message carries basic details about the flow and where it is deployed, including the names and values of some custom keywords which were defined in the Long Description field of the flow.

Once imported using the link on the right, the application ExampleKeywords is ready to be deployed and tested. To deploy, you can use a provided BAR file, as described on the Prepare tab.

Create an integration node owned server, inspect the message flow and then deploy it

Prepare the scenario by executing the following steps:

  1. Create an integration node by starting an App Connect Enterprise command console and executing the following commands:
    mqsicreatebroker TESTNODE
    mqsistart TESTNODE
    mqsicreateexecutiongroup TESTNODE -e server
  2. Back in the Toolkit, open the message flow Keywords.msfglow. Click on the flow's backdrop (as opposed to having any of the node's selected) and switch to the Properties tab, to display the node's properties. You should see:
    Short description:
    This message flow is designed to demonstrate how the keywords capability works
    Long description:
    $MQSIAuthor = Ben ThompsonMQSI$
    $MQSIEnvironment = DemoMQSI$
  3. Drag and drop deploy the provided BAR file named Keywords.bar which you will find inside the ExampleKeywords application.

Use the Flow Exerciser to test the tutorial scenario

  1. Open Keywords.msgflow.
    • Click the Flow Exerciser icon to start testing the flow (when challenged you do not need to redeploy the flow again).
    • Click the Send Message icon .
    • One saved input message named InputMessage is provided. Select it and click Send and your message is sent to the HTTPInput node.
    • InputMessage contains this data:
      The input data can be anything!
    • Click on the reply message and you will see that it contains this data:
      <?xml version="1.0" encoding="UTF-8"?>
      <Message>
        <NodeName>ACEv11005</NodeName>
        <ServerName>server1</ServerName>
        <ApplicationName>ExampleKeywords</ApplicationName>
        <FlowName>Keywords</FlowName>
        <Keywords>
          <Author>Ben Thompson</Author>
          <Environment>Demo</Environment>
        </Keywords>
      </Message>
    • After you close the dialog, the path taken through the message flow is highlighted.

This tutorial has shown how to assign custom keywords to message flows and then once deployed, read the keywords using the Java Integration API.
